# 业务流程说明

## 1. 线索转商机

- 线索录入 → 评估 → 转为商机 → 分配负责人

## 2. 商机推进

- 需求引导 → 方案阶段 → 商务谈判 → 合同审批 → 成交关闭
- 每个阶段关联预计金额、预计成交时间与跟进记录

## 3. 报价/合同/开票审批

- 提交审批 → 触发区域匹配流程 → 多级审批 → 结果回写
- 审批待办通过钉钉通知

## 4. 合同与回款

- 合同签署 → 开票申请 → 回款记录 → 业绩汇总

## 5. 跟进与待办

- 客户互动以“活动”记录（电话/会议/拜访/邮件）
- 需要后续行动的事项以“任务”记录并设置截止时间
- 任务按到期时间与负责人分发，支持完成/取消

## 6. 客户与联系人维护

- 客户建档 → 分配负责人 → 关联联系人 → 关联商机/合同/回款
- 联系人新增 → 关联客户 → 记录关键人/职位 → 跟进记录沉淀

## 7. 合同流程（参考销售易）

- 商机成交 → 新建合同 → 提交审批 → 审批通过 → 合同生效
- 合同变更/作废 → 触发二次审批（如启用）
- 合同生效后生成回款计划与开票申请

## 8. 开票申请

- 选择关联合同 → 填写开票信息 → 提交审批 → 开票完成

## 9. 审批流程

- 按区域匹配流程 → 节点审批 → 结果回写业务对象
- 审批待办通过钉钉通知（仅通知，不在钉钉内审批）

## 10. 钉钉组织与人员同步

- 触发方式：后台手动点击“同步钉钉用户与组织”（也可通过管理命令/API触发）。
- 组织拉取：拉取钉钉全量部门用于用户归属计算，但系统 `Region` 仅同步根部门与一级部门。
- 区域口径：根部门固定 `dept_id=1`；一级部门定义为 `parent_id=1`。
- 用户拉取：按全量部门遍历分页拉取，覆盖所有子部门成员。
- 用户归属：用户 `region` 按部门链回溯到一级部门；多部门时按第一个部门优先。
- 历史清理：每次同步会将历史二级及以下钉钉区域的业务引用迁移到一级区域（无法回溯则迁移到根部门）后删除。
- 手工区域：`dingtalk_dept_id` 为空的区域视为手工维护区域，不参与钉钉清理，可继续手工新增和维护。

### 10.1 同步流程时序图（文字版）

```text
管理员/定时任务
  -> CRM 同步入口（后台按钮 / 管理命令 / API）
  -> 获取钉钉 access_token
  -> 调用 listsubid 拉取全量部门 ID
  -> 按部门 ID 获取部门详情（构建部门父子关系）
  -> 过滤为根部门+一级部门，更新 Region
  -> 按全量部门 ID 分页拉取用户（含子部门）
  -> 用户去重（同一用户多部门合并）
  -> 按用户部门链回溯一级部门，写入 user.region
  -> 识别历史二级及以下钉钉 Region
  -> 迁移所有 Region 外键引用到一级区域（异常回溯到根）
  -> 删除历史二级及以下钉钉 Region
  -> 返回同步统计（部门总数/新增/更新，用户总数/新增/更新）
```
