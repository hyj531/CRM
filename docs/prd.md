# PRD：商机管理系统

**版本**：v1.1

## 1. 背景与目标

### 背景
公司需要统一的商机管理系统，用于商机、合同、开票与回款的全过程管理，并支持按区域公司分级权限。审批流程需要可配置，并通过钉钉做待办通知。

### 目标
- 建立统一的商机全流程管理能力：商机、客户/联系人、合同、开票、回款。
- 具备区域组织树与基于角色的数据范围权限。
- 支持报价、合同、开票审批流程的可视化配置与执行。
- 支持钉钉 SSO 与组织同步，审批待办通知可触达审批人。

## 2. 范围与边界

### 范围内
- 销售看板、商机、客户/联系人、合同、开票、回款。
- 审批流程引擎与流程配置界面。
- 基础报表与看板（漏斗、阶段分布、合同/回款汇总）。
- 区域权限与组织管理。
- 钉钉登录与组织同步、通知。

### 范围外
- 外部 BI/数仓、复杂预测算法。
- 电子签章与合同法务系统对接。
- 钉钉内直接审批动作（仅通知）。

### 运行环境与数据库
- 本地开发：使用 `.env.dev`，`DATABASE_URL` 指向 sqlite（`db.sqlite3`）。
- 生产环境：使用 PostgreSQL，优先通过 `DATABASE_URL` 配置；也可使用 `POSTGRES_DB/USER/PASSWORD/HOST/PORT` 组合配置。

## 3. 角色与权限

### 角色
- 销售经理
- 区域合伙人
- 区域负责人
- 集团合伙人
- 总部管理员

### 数据范围规则
- 默认：自己 + 下级区域
- 可配置：仅本人 / 本区域 / 本区域+下级 / 全局

### 常用文档权限口径（新增）
- 常用文档采用“目录级角色权限”，不沿用区域数据范围过滤。
- 权限维度：查看 / 下载 / 上传 / 编辑 / 删除（角色粒度）。
- 权限主体：角色；目录与权限配置仅管理员可维护。
- 可见范围：全公司共享，普通用户仅可见具备“查看”权限的目录与文档。

## 4. 业务模块

- 商机管理：阶段推进、金额与时间、停留天数、挂起、归档
- 客户/联系人：基础信息维护、关联商机
- 合同/开票/回款：关联商机与客户，审批后生效
- 活动/任务：待办提醒、跟进记录

### 活动管理（Activity）

活动用于记录**已发生**的客户互动（电话/会议/拜访/邮件），作为跟进记录的来源。

- 新增/编辑/查看/删除活动记录
- 活动类型：电话/会议/拜访/邮件（可扩展）
- 关联对象：线索/商机/客户/联系人/合同（至少支持线索/商机/客户）
- 关键字段：主题、类型、发生时间、负责人、备注
- 支持按时间、类型、负责人、关联对象筛选
- 支持按商机聚合展示近期活动
- 权限：按区域 + 负责人范围过滤


## 4.1 细化设计（参考销售易）

### 客户管理（Account）
定位：客户主数据中心，承载商机、联系人、合同、开票、回款的聚合入口。

页面结构（销售易风格）：
- 客户列表页：顶部统计卡 + 左侧筛选 + 右侧列表（卡片/表格）
- 客户详情页：顶部摘要条 + Tabs（商机/联系人/合同/回款/跟进）
- 新建客户：分区表单（基础信息/业务信息/归属信息）

核心功能：
- 客户建档、编辑、合并、归档
- 关联商机、联系人、合同、回款的聚合视图
- 客户级别与价值评级可配置

关键字段：
- 客户名称、所属区域、负责人
- 客户级别、行业、企业性质、规模
- 统一社会信用代码、地址、电话、官网

筛选与排序：
- 客户级别、行业、负责人、区域、最近跟进时间
- 最近更新、创建时间、商机金额排序

权限点：
- 按区域+负责人数据范围
- 可配置“仅本人客户不可见他人客户”

### 联系人管理（Contact）
定位：客户关键联系人池，支撑跟进与合同沟通。

页面结构：
- 联系人列表页：筛选 + 列表（客户/职位/联系方式）
- 联系人详情页：基础信息 + 关联客户/商机 + 跟进记录
- 新建联系人：基础信息 + 归属客户

核心功能：
- 维护联系人信息与职位角色
- 关联客户与商机
- 跟进记录快速入口

关键字段：
- 姓名、职位、手机号、邮箱、所属客户
- 是否关键人、偏好、备注

筛选与排序：
- 客户名称、职位、负责人、最近跟进时间

权限点：
- 按区域+负责人
- 允许跨区域查看（可配置）

### 合同管理（Contract）
定位：商机成交后形成合同的主流程对象，支持审批与回款计划。

页面结构：
- 合同列表页：顶部统计 + 筛选 + 列表（状态/金额/回款进度）
- 合同详情页：合同信息 + 审批记录 + 回款计划 + 开票
- 新建合同：基础信息 + 关联商机/客户 + 回款计划

核心功能：
- 合同创建、编辑、提交审批、作废
- 回款计划管理（期次/金额/时间）
- 关联合同文件上传（后续扩展）

关键字段：
- 合同编号、合同名称、关联商机、客户名称
- 合同金额、最终结算金额、签约时间、合同状态
- 负责人、所属区域、审批状态

筛选与排序：
- 合同状态、审批状态、客户、负责人、签约时间

权限点：
- 按区域+负责人
- 合同审批节点可见范围

### 应收款（Receivable）
定位：面向合同的应收款追踪视图，聚焦应收金额与重点催收合同。

页面结构：
- 入口：合同列表页内新增“应收款”标签
- 应收款列表：仅展示应收款 > 0 的合同，精简字段展示

核心功能：
- 应收款过滤：应收金额 > 0（应收 = 合同金额或当前产值 - 已回款）
- 重点催收标记：支持手动标记/取消标记
- 高亮显示：被标记合同在列表中高亮展示

列表字段（精简列）：
- 合同名称、客户、合同金额、已回款、应收款、负责人、区域、操作

筛选与排序：
- 继承合同页现有筛选条件
- 支持“只看重点催收”

权限点：
- 与合同一致（按区域+负责人）

### 回款管理（Payment）
定位：合同回款的执行与追踪。

页面结构：
- 回款列表页：筛选 + 列表（合同/期次/金额/状态）
- 回款详情页：回款信息 + 关联合同/开票
- 新建回款：关联合同/期次

核心功能：
- 回款登记、状态变更
- 回款与合同金额对账提示（差额提示）
- 回款计划执行进度展示

关键字段：
- 关联合同、回款期次、应收金额、实收金额
- 回款日期、回款状态、负责人、所属区域

筛选与排序：
- 回款状态、合同、客户、回款日期、负责人

权限点：
- 按区域+负责人

### 开票申请（Invoice）
定位：合同回款前的开票申请与审批入口。

页面结构：
- 开票申请列表页：筛选 + 列表（合同/客户/金额/状态）
- 开票申请详情页：申请信息 + 审批记录
- 新建开票申请：关联合同/回款期次

核心功能：
- 新建申请、提交审批、驳回、作废
- 关联合同/客户/回款计划

关键字段：
- 申请编号、关联合同、客户名称
- 开票金额、税率、开票时间、申请状态
- 负责人、所属区域、审批状态

筛选与排序：
- 申请状态、审批状态、合同、客户、申请时间

权限点：
- 按区域+负责人
- 申请人仅可查看本人申请（可配置）

### 常用文档（Knowledge/Docs）
定位：沉淀公司常用文档，按目录授权共享，统一上传、查看、下载与维护。

页面结构：
- 常用文档列表页：左侧目录区 + 右侧文档列表区。
- 目录权限配置：管理员入口，按角色配置目录权限矩阵。

核心功能：
- 单层目录管理：新建、重命名、删除（非空目录禁止删除）。
- 文档管理：上传、查看列表、下载、编辑（含替换文件）、删除。
- 下载采用后端强校验接口，禁止以直链作为权限判断依据。

关键字段：
- 目录：名称、排序、启用状态、创建人/创建时间、更新人/更新时间。
- 文档：标题、文件、原始文件名、备注、所属目录、创建人/创建时间、更新人/更新时间。

权限点：
- 目录级角色权限：查看 / 下载 / 上传 / 编辑 / 删除。
- 管理员拥有全量权限并可维护目录与权限配置。

### 审批流程（Approval）
定位：报价/合同/开票的统一审批体系，区域分支审批。

流程配置（销售易风格）：
- 流程列表：按业务对象 + 区域展示
- 流程编辑：节点、审批人、条件分支（金额/区域/类型）
- 默认流程 + 区域覆盖

流程执行：
- 提交审批、审批通过、驳回、撤回
- 审批记录可追溯
- 钉钉待办提醒（含审批链接）
- 审批单支持钉钉手机端展示（同一链接自适应）

审批节点能力：
- 条件分支（金额区间/区域/客户级别）
- 并行/会签（后续扩展）
- 指定审批人/角色/部门

权限点：
- 发起人、审批人、管理员不同可见范围
- 审批记录不可修改

#### 公共流程引擎模块化
- 审批引擎抽为独立公共模块（Django app 级别）
- 业务对象通过“适配器注册”接入（合同/开票优先）
- 引擎提供统一 API：发起审批、审批流转、审批详情查询、待办推送
- 允许后续扩展新业务类型（回款/商机/其他）

#### 审批待办与表单展示（合同/开票优先）
待办通知：
- 使用钉钉待办能力推送审批任务（非普通消息）
- 待办链接指向审批表单，可直接进行同意/驳回

审批表单（专用页面）：
- 展示关键字段与审批状态、发起人信息
- 支持填写审批意见并提交
- 移动端（钉钉 App）同一链接自适应展示

合同附件展示：
- 在合同审批表单中展示附件列表
- 支持预览与下载（图片/PDF/Office 文档）

## 6. 报表与看板

- 商机漏斗与阶段分布
- 合同金额与回款汇总
- 个人/区域业绩指标（基础视图）

### 6.1 销售看板指标口径

**通用规则**
- 数据范围：按角色数据范围（仅本人/本区域/本区域+下级/全局）过滤。
- 时间范围：支持可配置周期；仅定义业务日期口径。
- 金额口径：按记录金额字段统计（商机预计金额/合同金额/回款金额/开票金额）。

**时间字段（业务日期）**
- 商机：`created_at`
- 合同：`signed_at`
- 回款：`paid_at`
- 开票：`issued_at`

**指标口径**
- 商机数量：在数据范围 + 时间范围内的商机记录数（按商机创建时间过滤）。
- 商机转化率：商机成交数量 / 商机总数量。成交数量=阶段为“成交关闭/框架合作”的商机数；分母为 0 时结果为 0。
- 预计成交金额：同范围内商机 `expected_amount` 求和（空值按 0）。
- 概率成交金额：同范围内商机 `expected_amount × win_probability%` 求和（成交概率为空按 0）。
- 商机阶段分布：同范围内按“当前阶段”分组统计商机数。
- 商机漏斗（金额）：同范围内按“当前阶段”分组统计 `expected_amount` 求和。
- 合同总额：同范围内合同 `amount` 求和（按合同签署日期过滤）。
- 已回款：同范围内回款记录 `amount` 求和，状态仅计 `partial/paid`，排除 `planned`。
- 应收余额：基于合同范围汇总，按合同 `current_output`（优先）或 `amount` 作为基数，减去该合同已回款总额（状态同上，回款日期不受时间筛选）。
- 开票总额：同范围内开票 `amount` 求和（按开票日期过滤）。
- 个人/区域业绩：
  - 归属规则：按各记录自身的 `owner` / `region` 归属统计。
  - 指标：商机数、预计金额、合同金额、回款金额（口径同上）。

## 7. 非功能需求

- 性能：支持中小规模并发与数据量
- 可用性：核心功能可用性>99.5%
- 安全：JWT+Session 双认证；敏感操作审计
- 可维护性：模块化设计，支持迭代升级

- 查询统一按数据范围过滤
- 审批流程按区域匹配，默认可回退到全局流程
